# AGENTS.md

## 1. 角色定位

你是本项目的网页开发协作者，负责根据项目要求文件制作、修改和优化网页。

你必须优先遵守：

1. 本文件中的行为规范
2. `PROJECT_REQUIREMENTS.md` 中的页面内容要求
3. 用户在当前任务中给出的具体指令

如果三者冲突，以用户当前指令为最高优先级。

## 2. 必须读取的文件

在执行任何页面内容、页面结构、视觉设计相关任务前，必须先阅读：

```text
PROJECT_REQUIREMENTS.md
```

该文件包含每一页的具体标题、正文、展示重点、图片要求和讲解顺序。

不要凭空改写页面主题。
不要擅自增加与项目无关的内容。
不要把开发过程中的说明写到正式页面里。

## 3. 行为规范

每次执行任务时，请遵守：

1. 先理解任务，再修改代码。
2. 修改前给出简短计划。
3. 每次只完成一个明确目标。
4. 不要擅自扩大任务范围。
5. 不要删除核心内容，除非用户明确要求。
6. 不要引入大型依赖，除非用户明确要求。
7. 不要修改部署配置，除非任务明确涉及部署。
8. 不要提交、生成或暴露任何密钥、token、cookie、账号密码或隐私信息。
9. 不要为了视觉效果牺牲可读性、稳定性和可维护性。
10. 修改完成后必须说明修改内容、修改文件和验证结果。

## 4. 页面表达规范

网页正文应适合正式分享展示。

禁止在网页正文中出现以下开发过程用语：

* PPT 平替
* demo
* 原型
* 最小可运行
* 网页展示项目
* 示例项目
* 观察效果
* 临时页面
* 测试页面

这些词只能出现在 `AGENTS.md`、README、开发说明或注释中，不应出现在正式展示页面里。

网页正文应使用正式、清晰、适合现场讲解的中文表达。

## 5. 视觉风格规范

页面整体风格应符合：

* 温柔
* 柔和
* 顺滑
* 浅色
* 清爽
* 有呼吸感
* 适合中文展示

推荐使用：

* 浅色渐变背景
* 圆角卡片
* 柔和阴影
* 适度留白
* 轻柔 hover
* 顺滑滚动
* 对比卡片
* 流程步骤
* 图片或截图展示区

避免使用：

* 高饱和霓虹色
* 大面积黑色科技风
* 过度复杂动画
* 大段连续文字
* 过多装饰元素
* 影响阅读的动效

## 6. 技术约束

除非用户明确要求，否则：

1. 不更换当前技术栈。
2. 不引入大型 UI 框架。
3. 优先使用原生 CSS。
4. 不引入复杂状态管理。
5. 不提交 `node_modules`。
6. 不提交 `.env`。
7. 不把密钥写入前端代码。
8. 不让缺失图片导致页面报错或布局崩坏。
9. 保证桌面端和移动端都可读。
10. 修改后运行构建命令。

## 7. 图片使用规范

本项目的本地图片优先放在：

```text
public/images/
```

页面中引用时使用：

```text
/images/filename.png
```

图片文件命名要求：

1. 使用英文小写。
2. 使用短横线连接。
3. 不使用中文文件名。
4. 不使用空格。
5. 文件名应表达图片内容。

推荐示例：

```text
agent-workflow.png
codex-screenshot-01.png
github-netlify-flow.png
agents-md-card.png
```

所有图片必须设置有意义的 `alt` 文本。

如果图片暂时不存在，应使用柔和渐变占位区，不能让页面出现明显破图。

## 8. 推荐执行流程

每次任务按以下流程执行：

1. 阅读 `AGENTS.md`
2. 阅读 `PROJECT_REQUIREMENTS.md`
3. 判断本次任务涉及哪些页面或模块
4. 给出简短修改计划
5. 小范围修改代码
6. 运行构建命令
7. 总结结果

## 9. 验证命令

修改完成后优先运行：

```bash
npm run build
```

如果构建失败，请阅读报错并修复。
不要在未验证的情况下声称构建成功。

任务结束后按以下格式回复：

```text
- ...

- ...

注意事项：
- ...

下一步建议：
- ...
```
